Aurora-Panel
多用户中转与订阅管理系统 (Airport Management System)

系统定位 & 商业价值

Aurora-Panel 是一款专为高速网络中转服务商设计的高性能、高容错订阅管理与计费面板。解决的核心痛点是：高并发下的流控阻断、自动化的多客户端配置订阅生成、以及全自动的支付开通流程。

分布式架构 多协议支持 一键部署 防封安全

核心技术架构

前端用户/管理面板

基于 React/Vite 框架与 Tailwind CSS 构建，提供响应式移动端适配，包含完整的账单流、节点状态监控与工单系统。

后端计费 API 核心

采用 PHP Laravel 框架开发，处理套餐周期计费、三方支付回调接口，支持高频缓存（Redis）承载订阅请求。

节点同步守护进程 (Daemon)

采用 Go/Python 编写，定时从 API 拉取活跃用户 UUID 同步到代理内核（Xray / Hysteria 2 / TUIC），实现秒级流量结算。

系统亮点功能

  • 主流代理协议全覆盖：原生支持 VLESS Reality, VMess gRPC, Trojan 以及目前吞吐性能极佳的 Hysteria 2。
  • 内置订阅转换 (Subconverter)：无需依赖外部接口，主站自动生成适配 Clash, Shadowrocket, Surge 等客户端配置。
  • 微信/支付宝/数字货币聚合支付：支持三方易支付自动回调开通套餐，实现完全无人值守自动运维。
  • 精细化流控限速：可对单个节点、单个用户设置独立的限速、限流以及最大在线 IP 客户端限制。

节点同步核心代码示例 (Go/Python Daemon)

后端节点通过安全的 Handshake 密钥通信，定期拉取活跃用户配置并上报流量数据：

# 节点流量统计与授权校验守护进程示例
import time
import requests

PANEL_URL = "http://your-panel.com/api/v1/node"
NODE_KEY = "node_secure_auth_key_12345"
NODE_ID = 5

class NodeDaemon:
    def __init__(self):
        self.headers = {"X-Node-Key": NODE_KEY, "Content-Type": "application/json"}
        
    def sync_users(self):
        try:
            res = requests.get(f"{PANEL_URL}/users?node_id={NODE_ID}", headers=self.headers, timeout=5)
            if res.status_code == 200:
                return res.json().get("users", [])
        except Exception as e:
            print(f"Sync failed: {e}")
        return []

    def report_traffic(self, traffic_data):
        try:
            res = requests.post(f"{PANEL_URL}/traffic", headers=self.headers, json={"node_id": NODE_ID, "data": traffic_data}, timeout=5)
            return res.status_code == 200
        except Exception as e:
            print(f"Report failed: {e}")
        return False
